# 웹서버의 역사 2세대: 동적 HTML 전달(서버 2세대)

정적 파일 제공 중심이던 초기 웹서버에서, 서버가 HTML을 동적으로 생성해 전달하던 시기의 핵심 변화와 의미를 간단히 정리한다.

## 한 줄 요약

- 서버가 요청마다 HTML을 만들어 보내기 시작하면서, 웹은 "문서 배포"에서 "애플리케이션 응답"으로 확장됐다.

## 배경과 전환점

- 정적 HTML만으로는 사용자별/시간별/상태별 응답을 표현하기 어려웠다.
- 서버 로직이 HTML을 생성하는 방식이 도입되며 개인화와 데이터 기반 응답이 가능해졌다.

## 핵심 개념

- 템플릿 기반 HTML 생성: 서버에서 데이터를 주입해 완성된 HTML을 반환.
- 서버 사이드 렌더링(SSR): 브라우저는 결과 HTML만 받는다.
- 상태 중심 응답: 요청 파라미터, 세션, DB 결과에 따라 응답이 달라진다.

## 요청-응답 흐름(2세대)

1. 브라우저가 URL 요청
2. 서버가 로직 실행 및 데이터 조회
3. 템플릿에 데이터 주입해 HTML 생성
4. 브라우저에 완성된 HTML 반환

## 왜 중요한가

- 사용자 경험이 정적 문서에서 상호작용 기반으로 확장됨
- 웹 애플리케이션 개념이 자리 잡는 계기
- 이후 API 분리, 클라이언트 렌더링으로 진화할 기반이 됨
