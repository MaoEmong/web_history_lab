# 웹 서버 역사 3세대 (템플릿 엔진 / Model 1) 예제

이 프로젝트는 웹 서버 역사 중 3세대에 해당하는 템플릿 엔진(Model 1) 방식을 풍부하게 이해하기 위한 예제다.  
JSP 한 파일 안에서 요청 처리와 화면 렌더링이 함께 이루어지는 구조를 관찰하며, 동적 HTML 생성의 흐름과 한계를 살펴볼 수 있다.

## 시대적 배경과 특징

3세대는 정적 HTML 중심의 1세대와, 단순한 동적 생성이 강조된 2세대를 지나 **템플릿 엔진을 통해 화면을 조립하는 단계**를 의미한다.  
이 시기의 핵심은 다음과 같다.

- 서버가 HTML을 생성하고, 브라우저는 결과만 표시
- 템플릿 파일 내부에서 데이터 바인딩과 화면 구성이 동시에 진행
- 개발 속도는 빠르지만 역할 분리가 약해 유지보수가 어려울 수 있음

## 핵심 개념 요약

- 템플릿 엔진(Model 1): 요청 처리와 뷰 렌더링이 동일한 파일에 결합된 구조
- 서버 사이드 렌더링(SSR): HTML이 서버에서 생성되어 클라이언트로 전달됨
- JSP 스크립틀릿: 템플릿 내부에서 자바 코드를 직접 실행해 데이터를 출력

## 요청 처리 흐름(개념)

1. 브라우저가 URL 요청
2. 서버가 JSP를 실행
3. JSP 내부 로직이 데이터 계산/가공
4. HTML이 동적으로 생성되어 응답

## 구성

- JSP 페이지: `src/main/resources/META-INF/resources/index.jsp`
- Model 1 구조: 한 JSP 파일이 요청 처리와 화면 출력 역할을 동시에 수행

## 학습 포인트

- 정적 HTML과 서버 사이드 템플릿의 차이
- JSP 스크립틀릿을 통한 데이터 출력 방식
- 빠른 개발과 유지보수 비용 사이의 트레이드오프
- Model 1이 왜 Model 2(MVC)로 진화했는지에 대한 이해
