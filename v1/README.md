# 웹서버의 역사 2세대: 정적 HTML 전달

정적 HTML 파일을 웹서버가 그대로 전달하던 1세대 기술을 재현한 프로젝트다.  
파일 경로가 곧 URL이 되는 방식을 확인한다.

## 1세대 방식 설명
초기 웹에서는 서버가 HTML 파일을 그대로 내려주고, 브라우저가 이를 렌더링하는 방식이 일반적이었다.  
동적 처리나 템플릿 엔진 없이 정적 파일을 그대로 제공하니 구조가 단순하고 예측 가능하다.  
이 프로젝트는 그 단순함과 한계를 체험하는 데 목적이 있다.

## 핵심 흐름
요청한 URL 경로에 대응하는 파일을 찾고, 서버가 파일을 그대로 응답으로 내려준다.  
브라우저는 내려받은 HTML을 렌더링하고, 필요한 정적 리소스(이미지, CSS 등)를 추가로 요청한다.

## 구성
- `src/main/resources/static` : 정적 HTML 파일 위치
  - `index.html`
  - `a.html`
  - `b.html`
  - `bio/c.html`

## index.html이 필요한 이유
웹서버는 디렉터리 경로(`/`)로 접근될 때 기본 문서를 자동으로 찾는데, 그 표준 이름이 보통 `index.html`이다.  
즉, `http://localhost:8080/`로 접속해도 첫 화면이 열리도록 하는 진입점 역할을 한다.

## 접속 예시
- `http://localhost:8080/index.html`
- `http://localhost:8080/a.html`
- `http://localhost:8080/b.html`
- `http://localhost:8080/bio/c.html`

## 실습 포인트
- 서버는 파일을 가공하지 않고 그대로 내려준다.
- 파일 경로가 URL이 되는 단순한 구조를 확인한다.
- 페이지 추가는 파일을 그대로 복사하거나 새로 만들어 배치하면 된다.

## 한계와 확장 방향
동일한 레이아웃이나 메뉴를 여러 페이지에 반복해서 넣어야 하며, 변경 시 전체 파일을 수정해야 한다.  
데이터에 따라 화면이 달라져야 하는 요구에는 취약하므로 이후에는 동적 렌더링이나 템플릿 방식으로 발전한다.
